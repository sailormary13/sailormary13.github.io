CrewFinder

Summary

CrewFinder helps sailors find crew so that they can spend less time on logistics and more time out on the water. This app could be used by any sort of organization that wants to help its members organize group events, but this example was applied to the Hoofer Sailing Club in Madison, WI. Hoofers has over 1,000 members and many different types of boats that all have different crew requirements.

How it works

It can be difficult find other people to take out a sailboat with, so this program has users create events so that other users can sign up as crew. A user fills out certain information about the event they are creating and then the event is displayed chronologically on the events page.

Cool features

  • Each boat at Hoofers has a maximum number of crew allowed. This number was programmed in for each boat type and is displayed as “open spots” for each event. When a user signs up, the number of “open spots” decreases by one. Once “open spots” hits zero, no more users can sign up as crew.
  • A list of crew is displayed for each event, and users can easily remove themselves from the list with the Remove button. However the user can back-out out of the Remove action before it is finalized.
  • A user can easily edit or remove their event. Again, there is a back-out option for the user.

Details

I created this app for my PHP Web Development class in Spring 2018. My favorite part of projects like this is doing high-level design because I must anticipate and understand the back-end and front-end requirements of each feature and how they all relate and connect to create the project as a whole. I used:
